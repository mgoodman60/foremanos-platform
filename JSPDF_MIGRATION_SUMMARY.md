## Key Differences: jspdf vs @react-pdf/renderer

| Aspect | jspdf (Old) | @react-pdf/renderer (New) |
|--------|-------------|---------------------------|
| **Paradigm** | Imperative (pdf.text(), pdf.rect()) | Declarative (React components) |
| **Code Style** | Procedural with state variables (y position) | Component-based with props |
| **Layout** | Manual positioning with coordinates | Flexbox-based layout |
| **Reusability** | Function helpers | React components |
| **Type Safety** | Manual type casting | Strong TypeScript support |
| **Maintainability** | Harder to modify | Easier to update and extend |
| **Testing** | Difficult to unit test | Can test components separately |

## Migration Patterns Applied

### 1. Header/Footer Pattern
**Old (jspdf):**
```typescript
const addHeader = () => {
  pdf.setFillColor(249, 115, 22);
  pdf.rect(0, 0, pageWidth, 30, 'F');
  pdf.text('FOREMANOS', pageWidth - margin - 25, 8);
};
```

**New (@react-pdf/renderer):**
```tsx
const RoomSheetHeader: React.FC<{ data: RoomSheetData }> = ({ data }) => (
  <View style={styles.header} fixed>
    <Text style={styles.brandTitle}>FOREMANOS</Text>
    <Text style={styles.headerText}>{data.project.name}</Text>
  </View>
);
```

### 2. Table Pattern
**Old (jspdf):**
```typescript
const addTableRow = (cells: string[], isHeader: boolean) => {
  cells.forEach((cell, i) => {
    const x = margin + (i * cellWidth);
    pdf.text(cell, x + 2, y + 3);
  });
  y += rowHeight;
};
```

**New (@react-pdf/renderer):**
```tsx
const TableSection: React.FC<{ headers: string[]; rows: string[][] }> = ({ headers, rows }) => (
  <View style={styles.table}>
    <View style={styles.tableHeaderRow}>
      {headers.map((header, idx) => <Text key={idx} style={styles.tableCellHeader}>{header}</Text>)}
    </View>
    {rows.map((row, idx) => (
      <View key={idx} style={styles.tableRow}>
        {row.map((cell, cellIdx) => <Text key={cellIdx} style={styles.tableCell}>{cell}</Text>)}
      </View>
    ))}
  </View>
);
```

### 3. Conditional Rendering
**Old (jspdf):**
```typescript
if (data.finishSchedule.totalItems > 0) {
  addSectionTitle(`FINISH SCHEDULE (${data.finishSchedule.totalItems} items)`);
  // ... render content
}
```

**New (@react-pdf/renderer):**
```tsx
const FinishSchedule: React.FC<{ data: RoomSheetData }> = ({ data }) => {
  if (data.finishSchedule.totalItems === 0) return null;
  return (
    <View>
      <Text style={styles.sectionTitle}>FINISH SCHEDULE ({data.finishSchedule.totalItems} items)</Text>
      {/* ... content */}
    </View>
  );
};
```

## Performance Considerations

@react-pdf/renderer may have different performance characteristics:
- **Slightly slower** initial render (React reconciliation overhead)
- **Better for complex layouts** (flexbox vs manual positioning)
- **Smaller bundle size** overall (jspdf is ~600KB, @react-pdf/renderer is ~400KB)
- **Memory usage** comparable for typical reports
