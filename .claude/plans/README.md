# ForemanOS Plans

Persistent plan files for multi-session development work.

## Naming Convention
`YYYY-MM-DD-short-description.md` (e.g., `2026-02-06-agent-teams-strategy.md`)

## Statuses
- **draft** -- Still being designed, not yet approved
- **in-progress** -- Actively being implemented
- **completed** -- Done, kept for reference
- **paused** -- On hold, can be resumed later
- **abandoned** -- Decided not to pursue

## Usage
1. Copy PLAN_TEMPLATE.md to a new dated file
2. Fill in sections as you plan
3. Update status as work progresses
4. Plans persist across chat sessions -- reference by filename to resume
